| **Medium** | `POST` handler — `jwtMiddleware(req)` call | Invalid/malformed JWTs are silently downgraded to `null` (guest) instead of being rejected. A tampered token should arguably return `401`, not be treated as anonymous — this masks auth failures and allows unauthenticated spam. | Return `401` when `jwtResult instanceof NextResponse` if the JWT was present but invalid. | **Medium** | `POST` handler — `responseSummary` / `queryStr` / `messageText` stored in DB | User-supplied strings are truncated but **not sanitized** before DB storage. While parameterized queries prevent SQL injection, the stored values become a **stored-XSS vector** if the admin dashboard renders them without escaping. | Sanitize/escape HTML entities (`<`, `>`, `&`, `"`, `'`) before persisting, or enforce strict output-escaping at the admin rendering layer. |
| **Medium** | `POST` handler — no rate limiting | Endpoint accepts unlimited feedback submissions per client. No throttling or deduplication, enabling spam or abuse of both the JSONL file and DB inserts. | Add rate limiting middleware (e.g., per-IP or per-userId sliding window) or a dedup key on `messageId` to prevent duplicate feedback. |
| **Medium** | `POST` handler — `appendFile` / `mkdir` block | JSONL write has **no isolated try/catch**. If `appendFile` throws (disk full, permissions), the entire request returns 500 — but the DB insert was never attempted, so the feedback is fully lost. | Wrap the JSONL write in its own try/catch. On failure, still attempt the DB insert so at least one copy survives, or return a specific error that lets the client retry. |
| **Low** | `POST` handler — `body` type assertion | `req.json() as { … }` is a **type assertion** with no runtime validation. If the client sends unexpected shapes (e.g., `messageIndex` as a string), fields silently become `undefined` with no error feedback. | Use a runtime validator (e.g., Zod) to parse the body and return `400` with a descriptive message on invalid input. |
| **Low** | `POST` handler — `validFeedback.includes(feedback as FeedbackType)` | The `as FeedbackType` cast satisfies TypeScript but **hides** the fact that `includes` receives an `unknown` value. If `feedback` is an object or array, `includes` returns `false` and the code silently falls back to `"none"` — potentially masking client bugs. | Validate `typeof feedback === "string"` before the `includes` check, or use a Zod enum. | **Low** | `POST` handler — no request body size limit | No `Content-Length` or body-size check.

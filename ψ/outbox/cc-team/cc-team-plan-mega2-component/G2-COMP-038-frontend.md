| severity | location | issue | fix |
|----------|----------|-------|-----|
| **Critical** | `POST` handler (body reading) | `request.json()` consumes the request body stream. If `verifyCSRFToken(request)` also attempts to read the body (e.g., to validate a JSON CSRF token), the second read will throw a runtime error and crash the handler. | Read the body **once** and pass the parsed payload to `verifyCSRFToken` instead of the raw `request` object, or ensure `verifyCSRFToken` only reads headers/cookies. |
| **High** | `redirectUri` construction | The `redirectUri` is dynamically built from client-controlled headers (`host`, `x-forwarded-proto`) without an allow-list. A manipulated `Host` / `X-Forwarded-Host` header can cause ThaiD to redirect the authorization code to an attacker-controlled origin (open-redirect / SSRF risk). | Maintain an environment-based allow-list of valid redirect URIs. Validate the constructed URI against it before appending it to the IdP URL. |
| **High** | `authUrl` construction | The OAuth flow is missing PKCE (`code_challenge` / `code_verifier`). Without PKCE, the authorization code is vulnerable to interception, especially in mobile/WebView contexts (`isWebView`). | Generate a `code_verifier`, hash it to a `code_challenge=S256`, append both to `authUrl`, and store the verifier in the state/session cookie for verification in the callback route. |
| **Medium** | `catch` block | All errors are caught and returned as HTTP 500. A CSRF failure should yield 403, and a bad JSON body should yield 400, but they are indistinguishable here. | Inspect `error` types/codes in the catch block. Return **403** for CSRF failures, **400** for malformed JSON, and **500** only for unexpected server errors. |
| **Medium** | Cookie `domain` option | The production cookie domain falls back to a hardcoded `.dataxo.info`, which can leak cookies to untrusted subdomains or break in other environments (staging, review apps). | Remove the hardcoded fallback. Drive the cookie domain strictly from an environment variable, or omit the `domain` attribute unless explicitly configured. |

Redact or omit sensitive fields (`state`, `redirect_uri`, tokens) in production. |
| **Low** | `request.json()` call | The JSON body is parsed without checking `Content-Type` or catching malformed payloads, which can throw an unhandled exception before the try/catch if the stream is already consumed or invalid. | Wrap the JSON parse in its own validation block, or use a schema validator (Zod).

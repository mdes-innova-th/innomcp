# A11y / UX Audit — `CanvasPanel.tsx`

| Severity | Location | Issue | Fix |
|---|---|---|---|
| **Critical** | Outer `<div>` (root) | No landmark / region role wrapping a meaningful "Collaboration Canvas" surface → screen-reader users can't navigate to it via landmark shortcuts (D, R, etc.). | Wrap with `<section aria-labelledby="canvas-title">` (or `<aside>` if it's tangential) and put an `id` on the `<h3>` to reference. |
| **Critical** | Node-graph container `<div className="grid …">` | The visual node-and-edge graph is completely opaque to AT: 4 nodes with no semantic meaning, no text alternative, no description of edges/connections. | Provide an accessible alternative: (1) an `aria-label` on the container summarizing the graph, and (2) an `<ol>` / `<ul>` of nodes with roles/edges, either rendered as a visually-hidden "long description" or surfaced via a "View as list" toggle. Example: `<div role="img" aria-label="Graph with 4 agents: BigBoss → Builder → Reviewer → Debugger → BigBoss">` *plus* a sibling `<ul className="sr-only">` listing the same. |
| **Critical** | Each `<div>` node card | Non-interactive content rendered as `<div>`s with `hover:scale-105` (implies interactivity) but no `tabindex`, no `role`, no `onClick` / `onKeyDown`, no `aria-label`. Either make them real interactive controls (drill-down to agent) or strip the hover affordance. | If interactive: `role="button" tabIndex={0} aria-label={`${node.label}, role ${node.role}`} onKeyDown` handler for Enter/Space, plus visible `:focus-visible` ring (currently no focus style at all → keyboard users get no feedback). If decorative, remove `hover:scale-105` and mark `aria-hidden`. |
| **Critical** | Node graph (visual layout) | **Screen-reader trap risk**: the per-node `<span className="animate-ping">` "alive" dot is `aria-hidden`-less and pulses infinitely. Some SR users mute animation via `prefers-reduced-motion` but the visual ping combined with no semantic value adds noise; combined with the `tabIndex={0}` recommendation above, an unbounded focusable list inside a landmark without a "skip" is a navigation trap. | Add `aria-hidden="true"` to the pulsing dot. Wrap the node list in a container with a skip link ("Skip node graph") and ensure tabbing order has a single sensible stop (one focusable group container using `role="toolbar"` or `role="listbox"` rather than 4 focus stops). |
| **Critical** | `<h3>` and `<h5>` | Heading order is broken: starts at `<h3>`, then jumps to `<h5>` ("ลำดับการสื่อสารปัจจุบัน"). Skips `<h4>`, confuses heading navigation (`1`–`6` outline). | Use `<h3>` for the panel title, then `<h4>` for the connection list subheading (or a `<h2>` at page level if this is a top-level region). |
| **High** | Color tokens (e.g. `text-sky-400`, `text-violet-400`, `text-amber-400`) on `bg-{color}-500/10` | **Color contrast risk**: `*-400` foregrounds at small sizes (`text-[9px]`, `text-[10px]`, `text-[11px]`) over translucent dark backgrounds frequently fail WCAG AA 4.5:1 (large) / 7:1 (normal). The 9-10px role/header text is well below AA. | Increase to `*-300` *or* `*-200` for foreground, darken background opacity to `/30`+, and bump smallest sizes to at least 12px. Verify with Stark/Axe. For role label at 9px, that is unreadable at any contrast — increase to ≥12px. |
| **High** | Node cards | Color (`border-sky-500` etc.) is the **sole** differentiator between agents (Orchestrator / Features / Auditing / Time-Travel). Color-blind users lose the role cue entirely; the tiny 9px text is not a redundant signal at legible sizes. | Add an icon or shape glyph per role (e.g. a small SVG inside the card) with `aria-label` on the icon, and a visible text label that's ≥12px. |
| **High** | Connection rows (`<div className="flex items-center justify-between …">`) | Plain `<div>`/`<span>` markup with no list semantics and no semantic relationship between `from`, `action`, `to`. AT reads three unrelated fragments. | Render as `<ol>` of `<li>`, with structured text: *"Step 1: BigBoss assigns task to Builder"* (not "BigBoss ↳ action ↳ Builder"). Use `<dl>` if you want label-value pairing. |
| **High** | Connection text | The `↳` arrows convey direction visually only; visually impaired/keyboard-only users still get the text, but the decorative glyphs read as "right arrow" or silently in some SRs and add visual noise. | Replace with text ("→" rendered as an icon with `aria-hidden="true"`) and ensure the sentence is grammatically complete (e.g. "BigBoss → มอบหมายงาน → Builder"). |
| **High** | `animate-ping` dot | No respect for `prefers-reduced-motion`. Pulses for vestibular-sensitive users. | Add `motion-safe:animate-ping` (Tailwind), and provide a static `bg-emerald-500` fallback dot. Wrap with `aria-hidden="true"`. |
| **High** | Focus management | The component is fully **mouse-only**: `hover:scale-105` exists, but there is no `:focus-visible` style, no `tabindex`, and no `outline`/`ring` utility. Keyboard users cannot see focus on anything. | Add `focus-visible:ring-2 focus-visible:ring-primary focus-visible:ring-offset-2 focus-visible:outline-none` to node cards (if made focusable), and ensure the panel itself isn't a focus trap (no autoFocus, no rogue `tabIndex={0}` on a non-interactive container). |
| **Medium** | `<div className="flex-1 min-h-[350px] relative …">` | `overflow-hidden` + `flex justify-between` with a `min-h-[350px]` fixed-height graph — at 200% zoom or 320px viewport width the graph clips and the node labels (especially the 9px role) become unreadable. Also no responsive variant: 2-column grid becomes cramped. | Allow vertical scroll (`overflow-y-auto`), drop the `min-h-[350px]` in favor of `min-h-0` inside the flex parent, and switch to `grid-cols-1 sm:grid-cols-2`. |
| **Medium** | "Node Graph" container | Marked `relative` but no `id` / `role` for the graph itself; no `<title>`/`<desc>` for SVG if later you add edges. | If you add an SVG layer for edges, give it `role="img"` with `<title>` and `<desc>` referenced by `aria-labelledby`. |
| **Medium** | Connection list key | Using array index `key={i}` is fine here (static) but if connections ever become reorderable, it breaks SR announcement stability. | Use a stable id on each connection (`c.id` or derived hash). |
| **Medium** | Thai-language text mixed with English | Mixed-language content without `lang` attributes — screen readers may mispronounce Thai or skip it depending on default voice. | Wrap Thai strings in `<span lang="th">` (and English in `lang="en"` if any). Affects the description `<p>` and the connection actions. |
| **Medium** | "Collaboration Canvas" header | Not programmatically associated with the description (`<p>`) — SR users hear the heading then a separate "text" block. | Add `aria-describedby="canvas-desc"` on the heading and `id="canvas-desc"` on the `<p>`. |
| **Low** | Decorative dots/borders | Border tokens like `border-border/40` and `border-border/20` may not pass 3:1 against the background for non-text UI (WCAG 1.4.11). | Bump to `border-border` or ensure 3:1 in both light and dark themes. |
| **Low** | `<div className="text-[9px] …">{node.role}</div>` | 9px text is below WCAG minimum for legibility regardless of contrast; treats role as a footnote when it's actually the primary classifier. | Increase to ≥12px and consider promoting to a semantic label (`<strong>` or a `<dl>`). |
| **Low** | Missing alt / "image" semantics | No images, but the graph itself is a *visualization* and should be treated as one for AT. | Add a visually-hidden summary sentence: *"4 agents collaborating: BigBoss (Orchestrator) → Builder (Features) → Reviewer (Auditing) → Debugger (Time-Travel) → BigBoss."* Linked via `aria-describedby` on the graph container. |
| **Low** | No live region | If connections update in real time (the "alive" dot implies liveness), there's no `aria-live` announcement. | Add `<div aria-live="polite" aria-atomic="true" className="sr-only">` that announces connection changes. |

### Summary of risk profile
- **Critical** items are all about *programmatic meaning* (landmarks, graph semantics, focus) — i.e., a screen-reader or keyboard user currently perceives this panel as a wall of decorative text.
- **High** items are about *perceivability* (contrast, color-only encoding, heading order, reduced motion, list semantics).
- **Medium/Low** are resilience/robustness (zoom, i18n, live announcements, non-text contrast).

Fixing the **Critical + High** set moves this component from "visually polished but inaccessible" to roughly **WCAG 2.2 AA-compliant**; the remaining items harden it for full AAA and edge-case users.

## Accessibility & UX Audit – `AgentActivityTimeline.tsx`

| Severity | Location | Issue | Fix |
|----------|----------|-------|-----|
| **Critical** | `PhaseSection` – `<button>` (phase header) | Missing `aria-expanded` and `aria-controls`. Screen reader users cannot perceive the expand/collapse state or associate the button with its content. | Add `aria-expanded={expanded}` and `aria-controls={uniqueID}`. Give the `<ol>` a unique `id` (e.g., `phase-${group.phaseId}`). |
| **Critical** | `PhaseSection` – `<button>` arrow (▾/▸) | Arrow characters are presentational but rendered as text. Screen readers may read “black down-pointing triangle” etc. No accessible label describes the action. | Wrap the arrow in a `<span aria-hidden="true">` and add visually hidden text: `<span className="sr-only">{expanded ? 'Collapse' : 'Expand'} phase</span>` or use `aria-label` on the button. |
| **Critical** | `PhaseSection` – `<button>` label | The button’s only visible label is the arrow and phase title. Lack of clear accessible name (e.g., “Expand [phase name]”) for screen readers. | Add `aria-label` on button: `aria-label={(expanded ? 'Collapse' : 'Expand') + ' ' + group.phaseLabel}`. Or use the approach above. |
| **Serious** | `EventBadge` – emoji icon | Emoji used as the sole visual indicator of event type. Screen readers may announce “clipboard” etc., but not the actual event type. On small screens the `<span className="hidden sm:inline">` is `display: none`, so the event type text is completely hidden from assistive tech. | Replace `className="hidden sm:inline"` with a screen reader only class (e.g., `sr-only` or `visually-hidden`). Add `aria-label={event_type}` to the outer `<span>`. |
| **Serious** | `EventRow` (compact mode) – `<li>` | Only a `title` attribute is used on the `<li>`. `title` is not reliably read by screen readers; the event type is not conveyed when the badge text is hidden. | Add `aria-label={ev.event_type + ': ' + ev.content}` to the `<li>`. In compact mode, also ensure the `EventBadge` has an accessible name (as above). |
| **Serious** | `PhaseSection` – `<ol>` content region | No landmark or region label for the timeline content. Users relying on landmarks cannot jump directly to the timeline. | Wrap the entire timeline (or each phase) with `<section aria-label="Agent activity timeline">` or add `role="region" aria-label="Events for [phase name]"` to the `<ol>`. |
| **Moderate** | `PhaseSection` – collapsed content | When a phase is collapsed, the list is removed from the DOM (conditional rendering). Focus is not managed; if focus was inside the list, it may be lost. | Consider using CSS visibility toggle (e.g., `hidden`/`visible` with `aria-hidden`) to preserve focus. Alternatively, after collapsing, move focus to the button or a fallback element. |
| **Moderate** | Color contrast (dark mode) | Many badges use `bg-xxx-900/40` (40% opacity) over a dark background. The text uses `text-xxx-300`. The actual contrast depends on the underlying dark background and may fall below 4.5:1. | Verify contrast using tools (e.g., WebAIM contrast checker). If needed, adjust opacity to ≥60% or use solid background colors with sufficient contrast. |
| **Moderate** | `EventRow` – timeline spine | The vertical line is an absolutely positioned `<span>` with `aria-hidden="true"`. | **Moderate** | No keyboard navigation beyond phase toggles | Users cannot focus or interact with individual events. Optionally enable keyboard arrow navigation within the timeline. |
| **Minor** | `AgentActivityTimeline` – root element | The component has a `className` prop but no `role` or `aria-label`. If multiple timelines exist, they need distinguishable labels. | Add `role="region" aria-label="Agent activity timeline"` to the wrapper `<div>`. |
| **Minor** | `PhaseSection` – “Pre‑task” phase label | The phase header for the initial group (before any `phase.started`) is not rendered, so users never see the label “Pre‑task”.

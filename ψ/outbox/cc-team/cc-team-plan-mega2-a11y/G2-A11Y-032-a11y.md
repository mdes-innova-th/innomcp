Here is the accessibility and UX audit for the `ChatStickyActions` component. 

*(Note: Color contrast was evaluated and passes WCAG AA standards for both light and dark modes, so no contrast issues are listed.)*

| severity | location | issue | fix |
|---|---|---|---|
| **High** | Root `div` (Hidden state) | **Keyboard Trap / Invisible Focus**: `opacity-0` and `pointer-events-none` hide the bar visually and from the mouse, but the buttons remain in the keyboard tab order. `aria-hidden="true"` does **not** remove focusable elements from the tab order, causing keyboard users to tab into invisible elements. | Add the `inert` attribute (`inert={!visible}`) if using React 19, or apply `tabIndex={-1}` to all child buttons when `!visible`, or conditionally render the buttons instead of just hiding them. |
| **High** | Root `div` & Unread Badge | **Layout/Positioning Bug (UX)**: The unread badge uses `absolute` positioning (`-top-1 -right-1`), but the parent `div` lacks `relative` positioning. The badge will render relative to the wrong DOM ancestor, breaking the layout. | Add `relative` to the root `div`'s `className`. |
| **Medium** | "แชทใหม่" (New Chat) button | **Missing Focus Indicator**: Unlike the icon buttons, this text button lacks explicit `focus-visible` styles. It relies on default browser outlines, which are often inconsistent, clipped, or hard to see against the background. | Add `focus:outline-none focus-visible:ring-2 focus-visible:ring-blue-500 rounded` to its `className` to match the other buttons. |
| **Medium** | Root `div` (`role="toolbar"`) | **Incorrect ARIA Role / Missing Arrow Nav**: `role="toolbar"` implies arrow-key navigation (Left/Right) and roving `tabIndex` per WAI-ARIA authoring practices. Relying solely on standard Tab navigation violates this expectation. | Change `role="toolbar"` to `role="group"` (which allows standard tabbing), **OR** implement Left/Right arrow key navigation with a roving `tabIndex` pattern. |
| **Medium** | All buttons | **Small Touch Targets**: Icon buttons (`p-1`) and the text button (no padding) result in touch targets smaller than 44x44px. This fails WCAG 2.5.5 (Enhanced Target Size) and makes the UI difficult to tap on mobile devices. | Increase padding (e.g., `p-2` for icons, `px-2 py-1` for the text button) or use `min-h-[44px] min-w-[44px]` combined with flex centering. |
| **Medium** | `onClear` handler | **Lost Focus on Destructive Action**: Clearing the chat likely removes or resets DOM nodes. Without explicit focus management, focus will reset to the `<body>`, severely disorienting keyboard and screen reader users. | Ensure the parent component programmatically moves focus to a logical, safe element (e.g., the "New Chat" button or the main chat input) immediately after `onClear` executes. |
| **Low** | Unread Badge | **Missing Live Region & Context**: The badge is a static `<span>`. Screen readers won't announce count changes dynamically when it updates, and "99+" may be read awkwardly depending on the SR language engine. | Add `role="status"` and `aria-live="polite"`. Add a descriptive `aria-label` (e.g., `aria-label={unreadCount > 99 ? 'มากกว่า 99 ข้อความ���ี่ยังไม่ได้อ่าน' : \`${unreadCount} ข้อความที่ยังไม่ได้อ่าน\`}`). |
| **Low** | SVG Icons | **Missing Intrinsic Dimensions**: The SVGs rely solely on Tailwind CSS classes (`h-4 w-4`) for dimensions and lack explicit `width`/`height` attributes. This can cause layout shifts (CLS) or rendering issues if CSS is delayed or disabled. | Add explicit `width="16"` and `height="16"` (or `24`) attributes directly to the `<svg>` tags. |

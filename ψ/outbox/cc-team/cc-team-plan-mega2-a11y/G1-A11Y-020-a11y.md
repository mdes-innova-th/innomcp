# A11y / UX Audit — `ChatEmptyStateManager.tsx`

| Severity | Location | Issue | Fix |
|---|---|---|---|
| **Critical** | `offlineBanner` `<div>` (line ~30) | Non-semantic `<div>` for a status/alert message; `text-red-700` on `bg-red-50` is only ~4.0:1 contrast (borderline for normal text, fails WCAG AA for small text in many conditions). Screen readers won't announce a connection-loss status. | Use `<div role="status" aria-live="polite">` (or `role="alert"` if urgent). Add an `aria-label` like "Connection lost, running offline". Bump text to `text-red-800` (~5.5:1) and ensure background passes 4.5:1. |
| **Critical** | Outer wrapper `<div className="flex flex-col gap-6 …">` | Container has no landmark or label. Users landing on this empty state get no orientation (no `main`, no `region`, no heading). | Wrap in `<section aria-label="Chat empty state">` or `<main>`. Ensure a visible `<h1>`/`<h2>` is rendered (currently none — hero is visual). |
| **High** | Three `<div className="animate-fade-in">` wrappers around hero, quick actions, prompts | Animation-only wrappers are non-semantic and not exposed to AT. Users on `prefers-reduced-motion` will see `animate-fade-in` still firing unless the CSS keyframe honors the media query (not visible here — assume it doesn't). Stagger also relies on `animationDelay` inline style, which is ignored by screen readers and may cause focusable children to render before being announced. | Replace with semantic structure (`<header>` for hero, `<nav aria-label="Quick actions">` for quick actions, `<section aria-label="Suggested prompts">` for prompts). Add `@media (prefers-reduced-motion: reduce) { .animate-fade-in { animation: none; } }` and fall back to instant visibility. |
| **High** | `stateKey` remount via `<div key={stateKey} …>` | Forcing a remount via `key` blows away focus, scroll position, and any in-progress selection. Returning SR users lose context. No `aria-live` region, so the state change ("continue your conversation" → "ask anything") isn't announced. | Drop the `key` and toggle content conditionally. Add `aria-live="polite"` on the inner region and a visually-hidden status message ("Showing reduced prompt suggestions") that updates with `hasMessages`. |
| **High** | No `aria-label`/`aria-labelledby` on the region that swaps between full vs reduced | Screen-reader users navigating by landmark/region will hear "region" with no name, and the state swap is silent. | Add `aria-label` that updates ("Welcome suggestions" vs "Continue your conversation") or use `aria-labelledby` pointing at a heading in the hero/hint. |
| **High** | `providerMode` prop accepted but not used in this file | Dead prop is an a11y smell: downstream components may switch behavior in ways the empty state doesn't disclose (e.g., remote vs local model). If those branches differ in capability, SR users should be told. | Either consume it to render a visually-hidden note ("Using remote model — responses may be slower") or remove it from this component's API. |
| **Medium** | `offlineBanner` icon "⚠️" | Emoji is decorative but rendered as text and read aloud by some SRs ("warning sign"). Pure emoji also fails 3:1 non-text contrast in some renderings. | Wrap emoji in `<span aria-hidden="true">⚠️</span>` and add `aria-label` on the banner. Consider replacing with an inline SVG icon with `role="img" aria-label="Warning"`. |
| **Medium** | `reducedState` hint `<p className="text-slate-400">` | `text-slate-400` on white = ~3.0:1 — **fails WCAG AA** (4.5:1 required for normal text, 3:1 for large). The "or just ask anything…" hint becomes unreadable for low-vision users. | Use `text-slate-600` (~7:1) or `text-slate-700`. If a muted look is required, ensure font-size ≥ 18.66px bold or 24px to qualify as "large text" (3:1 still fails with slate-400, so go darker). |
| **Medium** | `flex flex-col gap-6` layout with no skip mechanism | If `ChatWelcomeHero` or `StarterPromptsGrid` contain long lists of buttons (likely), keyboard users must Tab through every suggestion to reach the textarea. The textarea exists (`textareaRef`) but is below the prompts in the DOM order — focus order may push it far down. | Add a "Skip suggestions" link as the first focusable element that moves focus to `textareaRef.current`. Ensure DOM order places the textarea before the suggestion grids, or use `tabIndex={-1}` + programmatic focus on the skip link's target. |
| **Medium** | Stagger `animationDelay: "0ms" / "100ms" / "200ms"` | If `prefers-reduced-motion` isn't honored by the `animate-fade-in` class, content is **invisible to motion-sensitive users** for up to 200 ms — and if the keyframe is `opacity: 0 → 1`, the content is removed from the accessibility tree during that window (Safari/VoiceOver behavior with `opacity: 0`). | Either use `visibility` + `transform` instead of `opacity`, or set `animation-fill-mode: both` and a fallback `opacity: 1` rule under `prefers-reduced-motion: reduce`. |
| **Medium** | No `lang` attribute on Thai-language strings ("⚠️ การเชื่อมต้องขัดข้อง…", "หรือถามได้เลย…") | The `<html lang>` is presumably English/auto; Thai strings will be mispronounced by SRs (English phonemes) and Thai screen readers won't engage. | Wrap Thai strings in `<span lang="th">…</span>`, or set `<html lang="th">` if the whole app is Thai, or set per-region `lang` on the components containing Thai copy. |
| **Low** | Component returns a `<div>` (not a React Fragment) at the root | Minor — adds an extra wrapper that breaks `display: contents` assumptions and creates an unnamed generic element in the AT tree. | If layout allows, return `<>` or a `<section>` (see Critical row). |
| **Low** | `textareaRef` is forwarded but no `aria-controls`/`aria-describedby` relationship is established here | The textarea's relationship to the suggestions below is implicit only. SR users don't know that activating a suggestion will populate the textarea. | In `ChatWelcomeHero`/`StarterPromptsGrid`, have buttons call `textareaRef.current?.focus()` after `onQuerySelect`, and expose `aria-describedby` on the textarea pointing to the suggestions region. |
| **Low** | `text-center` on the offline banner with a single line of text | Centered, short, status-style text is fine, but combined with the emoji and red palette, users with cognitive load or dyslexia may miss it. | Add a leading icon + bold prefix ("**Offline** — connection lost") to improve scannability. |
| **Info** | Component is a "client" orchestrator with no error boundary | If any child (`ChatWelcomeHero`, etc.) throws, the whole chat empties with no fallback message — a screen-reader trap of silence. | Wrap export in an `ErrorBoundary` that renders an `role="alert"` fallback ("Suggestions failed to load — you can still type a question"). |

### Top 3 fixes (highest ROI)

1. **Add `role="alert"` + Thai `lang` + AA-compliant red** to `offlineBanner` — currently fails contrast *and* is silent to AT.
2. **Honor `prefers-reduced-motion`** for the fade-in keyframes; staggered `opacity: 0` can hide content from AT trees during the delay window.
3. **Stop using `key` to force remount** and instead add a polite `aria-live` region that announces the full→reduced state change, so SR users aren't disoriented.

## Accessibility + UX Audit: `ComparePanel.tsx`

| Severity | Location | Issue | Fix |
|----------|----------|-------|-----|
| **High** | `<input>` in form | Missing programmatic label. Placeholder alone does not provide an accessible name for screen readers. | Add `aria-label="พิมพ์คำสั่งเปรียบเทียบ"` or associate a `<label>` with `htmlFor`. Example: `<label htmlFor="compare-input" className="sr-only">พิมพ์คำสั่งเปรียบเทียบ</label>` and `id="compare-input"` on the input. |
| **High** | Submit button (`<button>`) | Button text changes to `"..."` while loading. Screen reader users hear “dot dot dot”, which is meaningless. Loading state is not announced. | Keep button text static (e.g. “เปรียบเทียบ”) and add `aria-busy={loading}`. Use a hidden `<span className="sr-only">กำลังโหลด</span>` to indicate loading. Alternatively, use `aria-label="กำลังโหลด"` when loading. |
| **High** | Loading indicator `<div>` | The loading message appears/disappears without any live region. Screen readers will not announce the status change. | Wrap the loading indicator in a container with `role="status"` and `aria-live="polite"`. Example: `<div role="status" aria-live="polite" ...>`. |
| **Medium** | Result cards container | New results appear after loading, but no live region is used to announce them. Focus is not moved to the results, so keyboard users may not notice the update. | Wrap the results section in a `<div>` or `<section>` with `aria-live="polite"` and `aria-label="ผลการเปรียบเทียบ"`. Optionally move focus to the first result card after load using `ref` and `focus()`. |
| **Medium** | Emoji icons (⏱, ⚡, 💵, ⚔️) | Emojis are used as icons to represent time, tokens, cost, and a decorative sword. Screen readers may read the emoji name (e.g. “stopwatch”, “high voltage”) but do not convey the intended meaning for all users. | Wrap each emoji in a `<span role="img" aria-label="เวลา: ">` or add visually hidden text after the emoji. For the decorative sword, use `aria-hidden="true"`. Example: `<span role="img" aria-label="เวลาประมวลผล:">⏱</span>`. |
| **Medium** | Color contrast: `text-muted-foreground`, `text-emerald-500` | The muted foreground and emerald green may have insufficient contrast against the panel background (especially with transparency/opacity). Tailwind defaults often pass, but contrast ratios must be verified with real background colors. | Check contrast ratios using a tool (e.g. WebAIM). If failing, increase opacity/weight or use darker shades (e.g. `text-gray-600` instead of muted). Ensure green meets 3:1 (large text) or 4.5:1 (small text). |
| **Low** | Result cards use nested `<div>`s | The list of model responses is not marked up as a semantic list. This reduces screen reader navigation efficiency (no list landmark, no item count). | Use `<ul>` with role `list` (implicit) and `<li>` for each response card. Each `<li>` can contain the card content. This provides a list structure that screen readers can announce (e.g., “List of 3 items”). |
| **Low** | Form submission `handleSubmit` | The form is submitted on Enter key, which is correct. However, there is no visual or programmatic feedback if the user tries to submit an empty query (just returns early). No error message is communicated. | Add `aria-describedby` or a live region to announce validation errors. For empty input, set a descriptive message and use `aria-invalid="true"` on the input. |

### Additional Recommendations (not tabled)

- **Focus management on load**: After the simulated load completes, use `useRef` to focus the first result card (set `tabIndex={-1}` on the card and call `.focus()`). This helps keyboard users.
- **Loading button disabled state**: While loading, set `disabled` on the button to prevent duplicate submissions. When disabled, use `aria-disabled` or `disabled` with proper styling to ensure focus is managed.
- **Landmark roles**: The entire panel could be wrapped in a `<section aria-label="Model Comparison">` or `<nav>` (if intended as a navigation element) to provide a landmark for screen readers.
- **Motion sensitivity**: The `animate-pulse` on the loading indicator may cause discomfort for users with vestibular disorders. Provide a `prefers-reduced-motion` media query fallback or remove the animation.

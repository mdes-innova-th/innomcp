# Accessibility & UX Audit — `ChatConversationHeader.tsx`

| Severity | Location | Issue | Fix |
|---|---|---|---|
| 🔴 Critical | Action buttons container (`opacity-0 group-hover:opacity-100`) | **Hover-only visibility hides actions from sighted keyboard users.** Tabbing reaches invisible buttons with no visual indication. | Add `focus-within:opacity-100` to the container: `opacity-0 group-hover:opacity-100 focus-within:opacity-100`. Better yet, always show actions or use `@media (hover: hover)` for the hide pattern. |
| 🔴 Critical | Clear confirmation `<div>` | **Confirmation UI is not a dialog — no role, no focus trap, no focus management, no Escape-key dismiss.** Screen readers get no announcement; keyboard users can't dismiss with Escape; focus stays on the trigger button behind the confirmation. | Wrap in a `<div role="alertdialog" aria-label="ยืนยันการล้างประวัติ">`. Move focus to "ใช่" on open (`useEffect` + `ref`). Add `onKeyDown` Escape → `handleCancelClear`. Trap Tab within the two buttons. Return focus to the clear trigger on cancel/confirm. |
| 🔴 Critical | Clear confirmation `<span>ยืนยัน?</span>` | **Confirmation label is a dead `<span>` disconnected from the action buttons.** Screen readers announce it as orphan text with no programmatic association to the confirm/cancel buttons. | Replace with `<div role="group" aria-label="ยืนยันการล้างประวัติ">` wrapping the label + buttons, or use `aria-describedby` on the confirm/cancel buttons pointing to an `id` on the `<span>`. |
| 🟠 High | Root `<div>` | **Non-semantic landmark.** The header is a major page section with no semantic role — screen readers can't navigate to it via landmark shortcuts. | Change to `<header>` (or add `role="banner"` if it's the page-level header, or `role="region"` with `aria-label` if it's scoped to the chat). |
| 🟠 High | Title `<p>` element | **Title is a `<p>`, not a heading.** Screen reader users cannot navigate by headings (H-key) to find the conversation. | Change to `<h2 className="text-sm font-medium …">{displayTitle}</h2>` (pick appropriate level for the page outline). Add a visually-hidden level if the visual size must stay `text-sm`. |
| 🟠 High | Message count badge `<span>` | **Number read aloud with no context.** A screen reader announces "5" with no indication of what it counts. | Add `aria-label={`ข้อความ ${messageCount} ข้อความ`}` or wrap in `<span aria-label="จำนวนข้อความ">{messageCount}</span>`. Alternatively, use `role="status"` with descriptive text. |
| 🟠 High | Action buttons container `<div>` | **Group of related buttons has no `role="toolbar"` or `aria-label`.** Screen readers announce each button individually with no grouping context. | Add `role="toolbar" aria-label="เครื่องมือแชท"` to the container `<div>`. |
| 🟠 High | All three `<svg>` icons (`SearchIcon`, `ExportIcon`, `TrashIcon`) | **SVGs lack `aria-hidden="true"`.** Since the parent `<button>` already has `aria-label`, the SVG's inner text/path content may be announced redundantly or produce noise. | Add `aria-hidden="true"` to every `<svg>` element. |
| 🟡 Medium | Clear confirmation appearance | **No `aria-live` announcement when confirmation appears.** Screen reader users on the clear button won't know the UI changed. | Add an `aria-live="polite"` region (or rely on `role="alertdialog"` which implicitly live-regions), and ensure the confirmation container announces "ยืนยันการล้างประวัติ?" when it mounts. |
| 🟡 Medium | Clear button when `disabled` | **Disabled button is still in Tab order and can receive focus, but has no `aria-disabled` fallback explanation.** Some assistive tech ignores the HTML `disabled` attribute. | Keep `disabled` but also add `aria-disabled="true"` and `tabIndex={isStreaming ? -1 : 0}` to remove from tab order when disabled. |
| 🟡 Medium | Cancel button in confirmation | **No explicit text color in light mode.** Inherits `text-gray-600` from the parent, but the button sits on `bg-gray-100` — contrast is ~4.6:1, barely passing AA for small text. | Set an explicit darker text color (e.g., `text-gray-800`) on the cancel button for a safer contrast ratio. |
| 🟡 Medium | Badge `bg-gray-100 text-gray-600` (light mode) | **Contrast ratio ~4.6:1** — barely passes WCAG AA for normal text (4.5:1) and fails for the `text-xs` size if audited at the smaller threshold. | Use `text-gray-700` or `text-gray-800` on `bg-gray-100` for ≥5:1 contrast. |
| 🟡 Medium | `agentCount` prop | **Prop is declared but never rendered.** If it's meant to convey information (e.g., "3 agents active"), that content is invisible to all users. | Either render it (with accessible labeling) or remove the prop to avoid dead code / confusion. |
| 🟢 Low | Search & Export buttons | **No visual disabled state.** If `onSearch`/`onExport` are `undefined`, buttons still appear interactive with no feedback on click. | Disable or hide buttons when their callbacks are undefined; add `aria-disabled={!onSearch}` etc. |
| 🟢 Low | `title` prop fallback `"แชท"` | **Hard-coded Thai string with no i18n mechanism.** If the app supports multiple languages, this fallback won't adapt. | Use a translation function/localization context for the fallback string. |
| 🟢 Low | Confirmation buttons `px-1.5 py-0.5` | **Very small touch targets** (~20×20px estimated). Fails WCAG 2.5.5 / 2.5.8 (minimum 24×24px for pointer targets). | Increase padding to meet at least 24×24px (e.g., `px-3 py-1`). |

---

### Summary of Top-Priority Fixes

1. **Make actions keyboard-visible** — add `focus-within:opacity-100` (1 line change, massive impact).
2. **Convert confirmation to `role="alertdialog"`** with focus management, Escape key, and `aria-label`.
3. **Use `<header>` + `<h2>`** instead of `<div>` + `<p>`.
4. **Add `aria-hidden="true"`** to all SVGs inside labeled buttons.
5. **Label the badge** — `aria-label` with context, not just a bare number.

# A11y / UX Audit — `CollapsibleAgentWrapper.tsx`

## Summary
The component is a custom disclosure (collapse/expand) around a multi-agent stream. It uses a `<button>` with `aria-expanded` (good), but the panel region has no `id` linkage, no `role`/labelling, the height animation can cause **content clipping + scroll traps**, and there are several contrast, semantics, and i18n/locale issues.

---

## Recommended Pattern (sketch)

```tsx
<button
  type="button"
  aria-expanded={isExpanded}
  aria-controls="agent-panel-region"
  aria-label={accessibleSummary}        // e.g. "Toggle agent details — streaming, 3 agents"
  onClick={toggle}
  className="... focus-visible:ring-2 focus-visible:ring-ring"
>
  <StatusIcon state={status} />         {/* icon + visually-hidden text, not color-only */}
  <span aria-hidden="true">{summaryText}</span>
  <Chevron expanded={isExpanded} />
</button>

<section
  id="agent-panel-region"
  role="region"
  aria-label="Agent activity details"
  aria-hidden={!isExpanded}
  // when collapsed, also use the `inert` attribute (React 19 / polyfill)
  // so focus cannot land inside hidden content
  inert={!isExpanded ? "" : undefined}
>
  <MultiAgentPanel ... />
</section>

<div className="sr-only" aria-live="polite">
  {liveAnnouncement}
</div>
```

## Top 5 Must-Fix Before Ship
1. Link button ↔ panel with `aria-controls` + `id` (issue #2)
2. Make status distinguishable without color (issue #6)
3. Add `aria-live` announcement for streaming/done transitions (issue #10)
4. Fix `maxHeight: 600px` clipping / focus-into-hidden risk (issue #3)
5. Stop auto-collapsing when `done` — users miss the result (issue #5)

| severity | location | issue | fix |

| **High** | Outer `<div>` | **Missing ARIA role/label.** The component visually separates chat messages by date, but screen readers will skip over it or just read the text without understanding it's a section divider, losing conversational context. | Add `role="separator"` and `aria-label={label}` to the outer `<div>`. |
| **High** | `new Date(date)` | **Timezone shift bug (UX).** Parsing date-only strings (e.g., `"2023-10-01"`) via `new Date()` treats them as UTC midnight. This shifts the date to the previous day in local timezones behind UTC, causing incorrect "วันนี้"/"เมื่อวาน" labels. | Force local timezone parsing by appending `T00:00:00` if no time is provided: `new Date(date.includes('T') ? date : date + 'T00:00:00')`. |
| **Medium** | Inner `<span>` & `<hr>` | **Screen-reader trap / redundancy.** If the wrapper announces the label via `aria-label`, the inner text will be read out again. Additionally, the two `<hr>` elements are purely decorative lines, but screen readers will announce them as "horizontal rules", creating noise. | Add `aria-hidden="true"` to both `<hr>` elements and the `<span>`. (Alternatively, replace the `<hr>` tags with CSS `::before`/`::after` pseudo-elements). |
| **Medium** | `<span>` classes | **Color-contrast risk.** `text-gray-500` (`#6b7280`) on a white background barely passes WCAG AA (4.6:1). Combined with `text-xs` (12px), it is very difficult for low-vision users to read. | Change `text-gray-500` to `text-gray-600` (`#4b5563`, 7.1:1 contrast) or `text-gray-700` to ensure robust readability for small text. |
| **Low** | `new Date(date)` | **Unhandled invalid date.** If an unparseable string is passed, `new Date()` returns `Invalid Date`. `Intl.DateTimeFormat` will then output "Invalid Date", which is confusing for sighted users and poorly announced by screen readers. | Validate `parsedDate` before rendering: `if (isNaN(parsedDate.getTime())) return null;` or provide a graceful fallback UI. |

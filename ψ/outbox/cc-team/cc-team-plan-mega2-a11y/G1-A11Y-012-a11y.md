## Accessibility & UX Audit: AIModelSelector.tsx

| Severity | Location | Issue | Fix |
|----------|----------|-------|-----|
| **Critical** | Listbox `<div role="listbox">` | No keyboard arrow-key navigation between options. Users cannot use ↑/↓ to move through options, violating WCAG 2.1.1 (Keyboard). | Add `onKeyDown` handler to listbox: ArrowDown/ArrowUp to move focus between options, Home/End for first/last, and track `activeIndex` state. Use `aria-activedescendant` pointing to the focused option's `id`. |
| **Critical** | Listbox options `<button role="option">` | Options lack unique `id` attributes, making `aria-activedescendant` impossible to implement. Screen readers cannot announce the currently highlighted option. | Add `id={`ai-mode-option-${mode.id}`}` to each option button. |
| **Critical** | Trigger `<button>` | Missing `aria-controls` attribute to associate the button with the listbox it controls. Screen readers cannot convey the relationship. | Add `aria-controls="ai-mode-listbox"` to the trigger button and `id="ai-mode-listbox"` to the listbox div. |
| **Critical** | Menu open state | Focus does not move into the listbox when opened. Keyboard users must Tab through the backdrop button before reaching options, creating a confusing tab order. | After `setIsOpen(true)`, use `useEffect` or `requestAnimationFrame` to focus the first option or the listbox container. Alternatively, implement full focus trap within the menu. |
| **High** | Escape key handler | When Escape closes the menu, focus is not explicitly returned to the trigger button. The `setIsOpen(false)` doesn't call `buttonRef.current?.focus()`. | Add `buttonRef.current?.focus()` in the Escape key handler after `setIsOpen(false)`. |
| **High** | Listbox header section | The header (title, description, `<kbd>`) is inside the `role="listbox"` container. Screen readers may announce this non-option content as part of the listbox, causing confusion. | Move the header outside the `role="listbox"` div, or wrap only the `<ul>` in the listbox role. Use `aria-describedby` on the listbox to reference the description. |
| **High** | Color contrast: `text-muted-foreground/85`, `text-foreground/85` | The `/85` opacity modifier reduces text opacity to 85%, which likely drops contrast below WCAG AA 4.5:1 requirement for normal text, especially on light/dark backgrounds. | Remove the `/85` opacity modifier or use solid colors that meet contrast requirements. Test with a contrast checker against actual background colors. |
| **High** | Small text sizes: `text-[11px]`, `text-[12px]` | Text at 11px and 12px is considered "small text" under WCAG and requires 4.5:1 contrast ratio (or 3:1 if bold and ≥14px). Current sizes with muted colors likely fail. | Increase minimum font size to 14px for body text, or ensure contrast ratio is at least 4.5:1. Consider `text-[13px]` minimum for helper text. |
| **Medium** | Backdrop `<button>` | The full-screen backdrop button is focusable and has `aria-label="ปิดเมนูโหมด AI"`, but it's essentially a click-outside handler. Keyboard users may accidentally focus it and be confused by its purpose. | Add `tabIndex={-1}` to make it non-focusable, or replace with a non-interactive `<div>` with `onClick` handler. Ensure Escape key still closes the menu. |
| **Medium** | Trigger button accessible name | The button's accessible name is derived from its text content ("AI" or `current.short`), which doesn't clearly convey it's a menu trigger for selecting AI processing mode. | Add `aria-label="เลือกโหมดการประมวลผล AI: ${current.label}"` to provide a clear, descriptive accessible name. |
| **Medium** | Selected option visual indicator | The selected state relies on color (`selectedTone`) and a ring, but there's no text indicator like "✓ Selected" or `aria-label` update. Colorblind users may not perceive the selection. | Add a checkmark icon (like `faCheck`) next to the selected option with `aria-hidden="true"`, and ensure `aria-selected="true"` is properly announced. Consider adding "(เลือกแล้ว)" text for screen readers. |
| **Medium** | Loading state during `handleModeChange` | No visual or ARIA indication that a mode change is in progress. Users may click multiple times or think the action failed. | Add a loading state: disable options during fetch, show a spinner, and add `aria-busy="true"` to the listbox during the request. |
| **Medium** | Global shortcut `Ctrl+M` | `Ctrl+M` may conflict with browser/OS shortcuts (e.g., minimize window on some systems). The shortcut is also not discoverable without opening the menu. | Consider using a less common shortcut like `Ctrl+Shift+M`. Add a visually hidden `<span>` with the shortcut info near the trigger, or document it in a help section. |
| **Low** | Menu positioning: `bottom-full` | The menu opens upward (`bottom-full right-0`), which may be clipped if the trigger is near the top of the viewport. No dynamic positioning logic. | Implement dynamic positioning: detect available space and open downward if insufficient space above. Use a library like `@floating-ui/react` for robust positioning. |
| **Low** | Icon-only content on mobile | On small screens (`sm:hidden`), only "AI" text is shown without the mode name. This may be insufficient context for users. | Ensure the `aria-label` or `title` provides full context. Consider showing the icon + "AI" with a tooltip on hover/focus. |
| **Low** | `<kbd>` element styling | The `<kbd>` element uses `text-[10px]` which is extremely small and may be unreadable for users with low vision. | Increase to at least `text-[12px]` and ensure sufficient contrast. |

### Summary

**Critical Issues (4):** Missing keyboard navigation, missing ARIA relationships, focus management failures.

**High Issues (4):** Color contrast violations, semantic structure problems.

**Medium Issues (5):** Discoverability, loading states, visual indicators.

**Low Issues (3):** Positioning, responsive design, minor styling.

### Recommended Priority Fixes

1. **Implement full keyboard navigation** (Arrow keys, Home/End, Enter/Space) with `aria-activedescendant`.
2. **Add missing ARIA attributes**: `aria-controls`, `id` on listbox and options, `aria-label` on trigger.
3. **Fix focus management**: Move focus into menu on open, return to trigger on close.
4. **Restructure listbox**: Move header outside the `role="listbox"` container.
5. **Fix color contrast**: Remove opacity modifiers on text, increase font sizes.

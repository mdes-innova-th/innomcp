| **High** | Root `<div>` | **Non-semantic markup & Keyboard Nav:** Using a `<div>` with `role="button"` and custom `onClick`/`onKeyDown` to trigger a file input is an anti-pattern. | **High** | Error `<p>` element | **Screen-reader trap (Missing Live Region):** Error messages are injected into the DOM dynamically but lack ARIA live region attributes. Ensure the hidden input uses a visually-hidden class (not `display: none`) so it can natively receive focus. |
| **Medium** | `<input type="file">` | **A11y (Hidden Input & Accessible Name):** The input uses `className="hidden"` (`display: none`) and `aria-hidden="true"`. This removes it from the accessibility tree, breaking label association and preventing screen readers from identifying it as a file upload control. | Remove `aria-hidden="true"`. Replace `className="hidden"` with a visually-hidden utility (e.g., `sr-only` or `absolute w-px h-px opacity-0`) so it remains in the a11y tree.

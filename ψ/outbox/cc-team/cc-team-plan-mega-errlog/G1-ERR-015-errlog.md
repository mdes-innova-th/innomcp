**Summary of top remediation priorities**

1. **Wrap every normalizer in `try/catch`** and emit a structured `warn` log with `{ domain, toolName, query, err }`. The current code lets `JSON.stringify` / `String()` / shape mismatches crash the request path.
2. **Add a `safeStringify` helper** — three `JSON.stringify` sites will silently produce `undefined` on circular refs, yielding empty-content facts that propagate to the answer.
3. **Introduce a `logger` (pino/winston)** — the file has zero observability; combined with missing error context, production debugging is currently impossible.
4. **Sanitize `raw` payloads** before storing them on `RetrievalFact`; the `evidence` domain in particular is likely to contain PII that must not leak into composed answers or logs.
5. **Detect upstream error shapes** (`toolResult.error`, `{ ok: false }`) and short-circuit with a logged warning rather than normalizing them as healthy facts.

| severity | location | issue | fix |
|---|---|---|---|
| **Critical** | `ThaiGeoToolOutput` | **MCP protocol violation:** Tool results must conform to the SDK’s `CallToolResult` shape (`{ content: ContentItem[], isError?: boolean }`). The raw discriminated union bypasses the required `content` wrapper, so any implementation returning this object directly will cause client-side deserialization failures or be dropped by the transport layer. | Derive tool output from `CallToolResult`. Wrap the JSON payload in `content: [{ type: "text", text: JSON.stringify(payload) }]` and set `isError` appropriately. |
| **High** | `ThaiGeoToolError` | **Error envelope incorrect:** Encoding domain-level failures (`NOT_FOUND`, `LOW_CONFIDENCE`, `DB_ERROR`) as a return value with `success: false` breaks MCP semantics. The transport layer treats these as successful RPC completions, breaking client retry logic, logging, and standard `McpError` handling. | Throw `McpError` (with `ErrorCode.InternalError` / `InvalidRequest`) for operational faults. For business-rule failures, return a `CallToolResult` with `isError: true`. Remove the `success` discriminant entirely. |
| **High** | `ThaiGeoToolInput` / `ThaiGeoToolContext` | **Missing timeout / cancellation contract:** No `AbortSignal` or deadline parameter is declared, so implementations have no way to propagate client cancellation or enforce query timeouts. This guarantees resource leaks (DB connections, HTTP sockets) and hanging promises when the MCP client disconnects mid-flight. | Add `signal?: AbortSignal` to `ThaiGeoToolInput` (or context). Pass the signal through to all downstream async geo/knowledge queries and DB drivers. |
| **Medium** | `ThaiGeoAttributes` | **Unhandled rejection / serialization crash:** The `[key: string]: unknown` index signature permits non-serializable runtime values (functions, `BigInt`, circular references). When the MCP server JSON-serializes the tool result, these values will throw unhandled `TypeError`s inside the transport/JSON.stringify layer. | Replace `unknown` with a strict recursive `JSONValue` type, or remove the index signature and explicitly whitelist allowed attribute fields. |
| **Medium** | `ThaiKnowledgeEntity` | **Race condition / stale-read hazard:** `version` and `updated_at` are optional loose `string` types with no format/revision constraints. Concurrent tool executions or cached responses may race or return stale data because the type lacks a required, comparable revision token for optimistic concurrency or cache coherency. | Make `version` required and type it as `number` (monotonic revision) or a branded `ISO8601String`. Apply the same requirement to `ThaiGeoEntity`. |
| **Low** | `ThaiGeoToolSuccess.data` | **MCP edge case – empty vs. missing ambiguity:** `data: ThaiGeoResultItem[]` allows an empty array while `success: true`. This collides semantically with the `NOT_FOUND` error code, leaving it ambiguous whether an empty array is a valid geo result or a silent failure that should have been an error envelope. | Narrow the success type to a non-empty array: `data: [ThaiGeoResultItem, ...ThaiGeoResultItem[]]`, forcing the implementation to emit `ThaiGeoToolError` (or `isError: true`) when zero matches are found. |

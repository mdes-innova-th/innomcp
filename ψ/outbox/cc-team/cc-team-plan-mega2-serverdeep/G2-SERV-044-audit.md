# Deep Bug Audit — `tmdTools.ts`

| severity | location | issue | fix |
|---|---|---|---|
| 🔴 Critical | `requireTmdAuthForTier` — `console.log` line | **Credential leak in logs**: `uid=${uid}` writes the real uid in plaintext to stdout/logs. An operator-grade log should never emit real credentials. | Remove uid value from log or redact: `uid=${uid.slice(0,2)}***`. Use `logBoth("info", …)` which already redacts, instead of raw `console.log`. |
| 🔴 Critical | `fetchWithTimeoutAndSignal` (truncated) | **External AbortSignal not merged with timeout signal**: If the caller passes an external `signal` (e.g. MCP request cancellation), the `AbortController` created for timeout is independent — the external signal never aborts the in-flight fetch. This causes **unresponsive tool cancellation** and wasted connections. | Use `AbortSignal.any([externalSignal, controller.signal])` (Node ≥16.14) or subscribe `externalSignal.addEventListener("abort", () => controller.abort())` inside the function. |
| 🔴 Critical | `fetchWithTimeout` / `fetchWithTimeoutAndSignal` | **Response body stream never drained on error paths**: If the caller checks `response.ok === false` and returns early without calling `.text()` / `.json()` / `.body.cancel()`, the underlying TCP socket is held until the stream GCs — a **resource leak** that exhausts the connection pool under load. | Always drain or cancel the body: `if (!res.ok) { res.body?.cancel(); throw … }`. Wrap in a helper that guarantees cleanup. |
| 🟠 High | `withTmdAuthParams` | **Unhandled throw from `new URL(urlBase)`**: If `urlBase` is malformed or relative, `new URL()` throws a `TypeError`. This propagates as an unhandled rejection if the caller doesn't wrap in try/catch. | Wrap `new URL()` in try/catch and re-throw with a descriptive `TMD_URL_INVALID` error envelope. |
| 🟠 High | `requireTmdAuthForTier` | **Synchronous throw inside async tool handlers**: When called at the top of an MCP tool handler, a thrown `Error` (e.g. `TMD_API_PARAMS_MISSING`) may bypass the MCP SDK's error envelope if the handler isn't `async`-wrapped correctly, producing an **unhandled rejection** that crashes the process or returns a raw JSON-RPC error without `structuredContent`. | Return `{ ok: false, error: … }` envelope instead of throwing, or ensure every tool handler wraps calls in try/catch that maps to the MCP error shape. |
| 🟠 High | Cache layer (`tmdCacheGet`/`tmdCacheSet`) | **Thundering herd / stampede on cache miss**: Multiple concurrent MCP calls for the same key all see a cache miss, all fire parallel HTTP requests to TMD, wasting rate-limit quota and amplifying latency. | Implement request coalescing: on cache miss, store a `Promise` in a temporary map so subsequent callers `await` the same in-flight request. |
| 🟠 High | `fetchWithTimeout` | **No retry on transient network errors**: A single `ECONNRESET`, `ETIMEDOUT`, or 5xx response immediately fails. TMD endpoints are known to be slow (20–31 s); transient failures are common. | Add configurable retry (e.g. 2 retries with exponential backoff) for 5xx / network errors, respecting `Retry-After` headers. |
| 🟡 Medium | `EmptyArgsSchema` — `.passthrough()` | **Schema allows arbitrary extra properties**: `.passthrough()` lets any unknown keys through. If downstream code ever iterates over parsed args, it could process unintended data. | Use `z.object({})` (strict) or `.strip()` instead of `.passthrough()`. Unknown args should be rejected per MCP spec. |
| 🟡 Medium | `redactUrlForLog` — regex fallback | **Incomplete URL cleanup after param removal**: After deleting `uid`/`ukey`, the fallback regex can leave `?&`, `&&`, or trailing `?` in the URL string. The cleanup regex only handles `?\&` and `[?\&]$`, missing `&&`. | Add replacements: `s.replace(/&&/g, "&")`, `s.replace(/[?&]$/,"")`, and loop until stable. Or reconstruct from `URL.searchParams` in the try branch (which already works correctly). |
| 🟡 Medium | `isAuthFailLikePayload` | **False-positive auth-fail detection**: Joins all field values with ` | ` delimiter. If `payload.message` happens to contain "invalid" and `payload.code` contains "uid" in unrelated contexts, the combined string triggers a false auth-fail detection. | Check each field individually against auth-fail patterns rather than concatenating. Use word-boundary regexes (`\bauth\b`). |
| 🟡 Medium | `safeStringify` | **Silent error swallowing**: The `catch {}` block converts a `JSON.stringify` failure (e.g. circular ref) to `String(obj)`, which may produce `[object Object]` — useless for debugging and indistinguishable from valid output. | Log a warning in the catch block; return `{ text: "<unserializable>", truncated: false }` or include a `_serializeError` flag. |
| 🟡 Medium | `appendFormatJson` | **Regex matches `format=` inside URL fragment**: `/([?&])format=/` would match `?format=` in the query string correctly, but if the URL contains `#format=` in the hash, it would incorrectly skip appending. Edge case, but violates TMD API contract. | Use `new URL(url)` to parse and check `u.searchParams.has("format")` instead of regex. |
| 🟡 Medium | Error envelope consistency | **`structuredContent` shape not enforced on all error paths**: The doc comment says `{ ok, meta, data }` but several code paths return raw MCP error objects or throw without wrapping, leading to inconsistent response shapes that break MCP clients expecting `structuredContent`. | Create a `tmdResult(ok, data, meta)` helper and mandate its use on every return path. Add a unit test that asserts all tool outputs conform to the envelope. |
| 🟢 Low | `getInnomcpMode` | **`process.env.INNOMCP_MODE` read on every call**: No caching; repeated string ops on hot paths. Minor perf concern. | Cache the result in a module-level variable; invalidate on `process.env` change if needed. |
| 🟢 Low | `sanitizeSnippetForLog` | **Over-aggressive stripping**: Removes `{}` `""` `''` ` `` ` from all content, making debug logs of JSON payloads nearly unreadable. | Only strip control chars and newlines; keep structural punctuation. Or limit to first N chars without character removal. |
| 🟢 Low | `DEFAULT_TIMEOUT_MS = 60000` | **60 s timeout with no per-endpoint override**: Some TMD endpoints (e.g. `WeatherToday`) take 20–31 s, but lightweight endpoints (`Station/v1`) should fail fast at 10 s. A blanket 60 s timeout masks hangs. | Accept `timeoutMs` per tool registration; default to 30 s for fast endpoints, 60 s for known-slow ones. |
| 🟢 Low | `requireTmdAuthParams` | **Deprecated function still exported**: `requireTmdAuthParams` delegates to `requireTmdAuthForTier("api")` but has no `@deprecated` runtime warning, so callers may never migrate. | Add `console.warn("DEPRECATED: use requireTmdAuthForTier")` or remove and force migration. |

---

**Top 3 most impactful fixes to implement first:**

1. **Merge AbortSignals** in `fetchWithTimeoutAndSignal` — without this, MCP request cancellation is broken.
2. **Drain/cancel response body** on non-OK status — prevents connection-pool exhaustion under real load.
3. **Redact `uid` from `console.log`** — credential leakage in logs is a compliance and security violation.

# Deep Bug Audit: `docWriterTool.ts`

| Severity | Location | Issue | Fix |
|---|---|---|---|
| **CRITICAL** | `safePath` (line ~46) | **Path traversal sandbox escape is incomplete.** `path.basename(filename)` strips *some* traversal but allows absolute paths on Windows (`C:\foo.docx` → basename `foo.docx` is fine, but `\\server\share\foo.docx` and NUL bytes aren't handled) and, more importantly, **filename with extension `.md` vs `.docx` is not validated** — a file like `evil.exe` would be written with a different extension. Also no allow-list check. | Validate extension against `{docx,pdf,md}` allow-list first; reject if `getExt()` returns null. Reject if basename contains null bytes. Reject reserved Windows names. Consider `path.normalize` + verify resolved path starts with `STORAGE_ROOT + path.sep`. |
| **CRITICAL** | `docWriterToolSchema` (line ~58) | **MIME / extension mismatch not enforced.** Schema only `describe`s that filename "must end with .docx, .pdf, or .md" — but there is no `.refine()` validator. A caller can pass `filename: "report.txt"` and `buildPdf`/`buildDocx` will be skipped, but the file may still be written as `report.txt` (raw content), or worse, `getExt` returns `null` and behavior is undefined. | Add `.refine(f => getExt(f) !== null, { message: "filename must end with .docx, .pdf, or .md" })` and derive format internally from extension. |
| **HIGH** | Top-level IIFE (line ~28) | **Unhandled promise rejection on `mkdir` failure swallowed silently.** `catch {}` hides ENOSPC, EACCES, EROFS, or readonly filesystem errors. Tool will then fail later with a confusing `ENOENT` instead of a clear "storage init failed" error. | Log error; if it's a permission/space issue, rethrow so process exits fast at startup. At minimum `console.error` and exit non-zero. |
| **HIGH** | `buildPdf` (line ~119) | **No timeout / hang protection.** `doc.end()` is called synchronously after a synchronous render loop, but if `doc.text()` ever throws (e.g., bad font, huge content causing internal pdfkit bug), the `error` handler fires `reject` — however **listeners are attached AFTER `new PDFDocument()`**, and synchronous throws inside the render loop (e.g., on `block.rows[r].forEach` when `r` index hits undefined row in malformed input — though schema prevents it, runtime data could still cause NaN page math) would not be caught because no `try/catch` wraps the loop. | Wrap the entire render loop in `try { ... } catch (e) { doc.destroy(); reject(e); }`. Add `Promise.race` with a 30s timeout. |
| **HIGH** | `buildPdf` (line ~125) | **Resource leak on error.** If an exception is thrown mid-stream, the `PDFDocument` instance is never destroyed; file descriptors and the internal font/buffer pool remain held until GC. | On error, call `doc.destroy()` and clear `chunks` array. |
| **HIGH** | `buildDocx` (line ~93) | **No size limit on input blocks.** `blocks` can be arbitrarily large; `Packer.toBuffer` will load everything into memory → DoS via OOM. A 10,000-block request with large strings will cause the process to crash. | Add a soft cap (e.g., 1000 blocks, 1MB total text) and validate in `.refine()` or pre-render size check. |
| **HIGH** | `buildPdf` table renderer (line ~158) | **`Math.max(1, block.rows[0]?.length ?? 1)` divides by cellWidth but if all rows have *different* lengths, later cells in shorter rows are skipped, and longer rows overflow off-page with no clipping.** The loop iterates `block.rows[r].forEach` with a fixed `cellWidth` computed from row 0 only. | Use the max column count across all rows; validate `rows` is rectangular in `.refine()`; clamp x to page width. |

Or use `rowHeight = Math.max(22, doc.heightOfString(cell, ...))`. |

### Summary

- **2 CRITICAL** — both around filename validation and sandbox integrity. The sandbox is the only safety boundary between MCP input and the host filesystem; it must be airtight.
- **5 HIGH** — focus on DoS / resource exhaustion (`buildDocx` memory, `buildPdf` no timeout, table render bugs that corrupt output silently) and an undeclared `numbering` reference that breaks DOCX output for ordered lists.
- **6 MEDIUM** — MCP protocol hygiene: error envelopes, progress, cancellation, streaming vs buffering.
- **7 LOW / INFO** — defense-in-depth and cross-platform correctness.

**Top three to fix first:**
1. Extension allow-list + path validation (`safePath` + `docWriterToolSchema.refine`).
2. `try/catch` + `doc.destroy()` in `buildPdf` render loop.
3. Declare the `"ordered"` numbering definition in the `Document` config.

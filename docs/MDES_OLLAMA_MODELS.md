# โมเดล MDES Ollama

## วิธีเลือกโมเดล
คลิกที่ชื่อโมเดลในส่วนหัวของ MDES → เลือกรายการที่ต้องการ

## โมเดลหลัก

| โมเดล | ขนาด | เหมาะสำหรับ | ความเร็ว |
|-------|------|------------|---------|
| gemma4:26b | 26B | ภาษาไทยทั่วไป, เอกสาร, อธิบาย | ปานกลาง |
| qwen2.5:7b | 7B | คำถามเร็ว, ทักทาย, ข้อมูลเบื้องต้น | เร็ว |
| qwen2.5-coder:7b | 7B | เขียนและแก้ไขโค้ด | เร็ว |
| deepseek-r1:32b | 32B | วิเคราะห์ซับซ้อน, เปรียบเทียบหลายทาง | ช้า |
| deepseek-v3:latest | varies | งานทั่วไปสมดุล | ปานกลาง |

## คุณสมบัติแต่ละโมเดล

### gemma4:26b (แนะนำ)
- ให้ผลลัพธ์ภาษาไทยเป็นธรรมชาติที่สุด
- เหมาะสำหรับ: สรุปเอกสาร, อธิบายเนื้อหา, ตอบคำถามทั่วไป
- รองรับบริบทขนาดใหญ่ (context window สูง)

### qwen2.5:7b
- ตอบสนองเร็วที่สุด เหมาะกับงานที่ไม่ซับซ้อน
- เหมาะสำหรับ: ทักทาย, ถาม-ตอบสั้น, ค้นหาข้อมูลเบื้องต้น, แปลประโยคสั้น ๆ
- ประหยัดทรัพยากร

### qwen2.5-coder:7b
- ปรับแต่งสำหรับงานพัฒนาโปรแกรม
- เหมาะสำหรับ: เขียนโค้ด, แก้บั๊ก, อธิบายโค้ด, แปลงภาษาการเขียนโปรแกรม
- สนับสนุนหลายภาษา เช่น TypeScript, Python, JavaScript

### deepseek-r1:32b
- ทำงานแบบ reasoning (คิดทีละขั้นตอนก่อนตอบ)
- เหมาะสำหรับ: โจทย์คณิตศาสตร์, ตรรกะ, วางแผน, วิเคราะห์หลายปัจจัย
- ใช้เวลาประมวลผลนาน แต่ละเอียด

### deepseek-v3:latest
- รุ่นล่าสุดของ DeepSeek ประสิทธิภาพสูง
- เหมาะสำหรับ: งานทั่วไปที่ต้องการความแม่นยำ, การสร้างเนื้อหา, การวิเคราะห์
- ความเร็วปานกลาง ใช้ทรัพยากรพอสมควร

## การเพิ่มโมเดลเอง
กด ⚙️ (การตั้งค่า) → เพิ่ม Provider → กรอก Base URL และชื่อโมเดลที่ต้องการ  
ระบบจะดึงรายการโมเดลจาก Ollama endpoint นั้นมาให้เลือกใช้

### 💻 The Ultra-Modern Semantic Router Class (Code)

โค้ดนี้คือหัวใจสำคัญ มันทำหน้าที่เปลี่ยน "ข้อความ" เป็น "เวกเตอร์" แล้วคำนวณระยะห่าง (Cosine Similarity) เพื่อหาหมวดหมู่ที่ใช่ที่สุด พร้อมระบบ Cache เพื่อความเร็วระดับมิลลิวินาที

```typescript
import { Ollama } from 'ollama';
import { LRUCache } from 'lru-cache';

// 1. Configuration: ปรับจูนค่าความแม่นยำ (Threshold) และ Model
const CONFIG = {
  OLLAMA_HOST: 'http://127.0.0.1:11434',
  EMBED_MODEL: 'nomic-embed-text', // เบา เร็ว แม่นยำสูงสำหรับงานจัดหมวดหมู่
  SIMILARITY_THRESHOLD: 0.45,      // ถ้าความเหมือนต่ำกว่านี้ถือว่าไม่เข้าพวก (Unknown)
  HYBRID_WEIGHT: 0.3,              // น้ำหนักให้ Keyword (0.3) vs Semantic (0.7)
};

// 2. Data Structure: นิยาม Category แบบเดิมแต่เพิ่ม description เพื่อให้ AI เข้าใจบริบทลึกซึ้ง
interface ToolCategory {
  id: string;
  description: string; // สำคัญมากสำหรับ Semantic Matching!
  keywords: string[];
  embedding?: number[]; // ที่เก็บ Vector (Cache ไว้ใน RAM ตอน Start)
}

// ข้อมูล Categories (ตัวอย่างย่อ จากที่คุณมี)
const CATEGORIES: ToolCategory[] = [
  { id: 'database', description: 'Operations involving SQL, NoSQL, data retrieval, storage, schemas, and vector DBs.', keywords: ['sql', 'query', 'database', 'mongo'] },
  { id: 'file', description: 'File system operations, reading/writing files, parsing CSV/JSON/PDF, and storage management.', keywords: ['file', 'read', 'write', 'upload'] },
  { id: 'ai_ml', description: 'Artificial Intelligence tasks, LLM prompting, text summarization, translation, OCR, and RAG.', keywords: ['ai', 'gpt', 'llm', 'genai'] },
  { id: 'devops', description: 'Server management, Docker containers, Kubernetes pods, system logs, and resource monitoring.', keywords: ['docker', 'k8s', 'server', 'deploy'] },
  // ... ใส่หมวดอื่นๆ เพิ่มตรงนี้ ...
];

export class IntelligentRouter {
  private ollama: Ollama;
  private cache: LRUCache<string, string>; // Cache ผลลัพธ์: Query -> CategoryID
  private categories: ToolCategory[] = CATEGORIES;
  private isInitialized = false;

  constructor() {
    this.ollama = new Ollama({ host: CONFIG.OLLAMA_HOST });
    this.cache = new LRUCache({ max: 500, ttl: 1000 * 60 * 60 }); // เก็บ 500 คำล่าสุด 1 ชม.
  }

  // 🔥 INIT PHASE: เปลี่ยนคำอธิบายหมวดหมู่ให้เป็น Vector (ทำครั้งเดียวตอน Server Start)
  public async initialize() {
    console.log('🚀 Initializing Semantic Vectors...');
    const promises = this.categories.map(async (cat) => {
      const response = await this.ollama.embeddings({ model: CONFIG.EMBED_MODEL, prompt: cat.description });
      cat.embedding = response.embedding;
    });
    await Promise.all(promises);
    this.isInitialized = true;
    console.log('✅ Semantic Router Ready: All categories vectorized.');
  }

  // 🧠 CORE LOGIC: Vector Math (Cosine Similarity)
  private cosineSimilarity(vecA: number[], vecB: number[]): number {
    const dotProduct = vecA.reduce((sum, a, i) => sum + a * vecB[i], 0);
    const magnitudeA = Math.sqrt(vecA.reduce((sum, a) => sum + a * a, 0));
    const magnitudeB = Math.sqrt(vecB.reduce((sum, b) => sum + b * b, 0));
    return dotProduct / (magnitudeA * magnitudeB);
  }

  // ⚡ HYBRID ROUTING: เร็วสุดด้วย Keyword + ฉลาดสุดด้วย Semantic
  public async categorize(userQuery: string): Promise<string> {
    if (!this.isInitialized) await this.initialize();

    // Step 1: Cache Hit? (เร็วที่สุด <1ms)
    if (this.cache.has(userQuery)) return this.cache.get(userQuery)!;

    // Step 2: Keyword Exact Match (เร็วมาก) - ถ้าเจอ Keyword ชัดๆ เอาเลยไม่ต้องเรียก AI
    const keywordMatch = this.categories.find(c => c.keywords.some(k => userQuery.toLowerCase().includes(k)));
    if (keywordMatch) {
      this.cache.set(userQuery, keywordMatch.id);
      return keywordMatch.id;
    }

    // Step 3: Semantic Matching (ฉลาดล้ำ) - เรียก Ollama แปลง Query เป็น Vector
    try {
      const response = await this.ollama.embeddings({ model: CONFIG.EMBED_MODEL, prompt: userQuery });
      const queryVector = response.embedding;

      let bestCategory = 'general';
      let maxScore = -1;

      for (const cat of this.categories) {
        if (!cat.embedding) continue;
        const score = this.cosineSimilarity(queryVector, cat.embedding);
        // Console log เพื่อ Debug ดูค่าความมั่นใจของ AI
        // console.log(`Category: ${cat.id}, Score: ${score.toFixed(4)}`);
        
        if (score > maxScore) {
          maxScore = score;
          bestCategory = cat.id;
        }
      }

      // ถ้า Score ต่ำกว่า Threshold แสดงว่า AI ไม่มั่นใจ -> ส่งเข้า General หรือถามกลับ
      const finalResult = maxScore >= CONFIG.SIMILARITY_THRESHOLD ? bestCategory : 'unknown';
      
      // Cache ไว้ใช้ครั้งหน้า
      this.cache.set(userQuery, finalResult);
      return finalResult;

    } catch (error) {
      console.error('⚠️ Semantic Router Error (Fallback to keyword):', error);
      return 'general'; // Fail-safe
    }
  }
}

// --- Usage Example ---

//   console.log(await router.categorize("optimize docker image ให้หน่อย")); // -> 'devops'
// })();

```

### 🧠 Deep Dive: เทคนิคระดับเทพ 2026 (เพื่อความเป็นที่ 1 ในเอเชีย)

1. **ทำไมต้อง Vector Embeddings? (The Core Magic):**
Keyword Matching แบบเดิมมัน "โง่" ครับ มันมองแค่ตัวอักษร แต่ Vector แปลงความหมายเป็น "ตัวเลขในมิติ"
* *Input:* "พรุ่งนี้ต้องเอาร่มไปไหม"
* *Vector:* เข้าใจว่าประโยคนี้มีบริบทใกล้เคียงกับ "ฝน", "สภาพอากาศ", "เปียก"
* *Match:* มันจึงวิ่งไปหาหมวด `weather` ได้เอง โดยที่คุณไม่ต้องใส่ keyword "ร่ม" ไว้ใน list เลย นี่คือ Semantic Search ที่แท้จริง


2. **Model Selection (Nomic vs Llama3):**
* **อย่าใช้ Llama3/Mistral ในการทำ Router:** เพราะมันช้า (Latency สูง) และเปลือง VRAM โดยใช่เหตุ
* **ใช้ `nomic-embed-text` หรือ `mxbai-embed-large`:** โมเดลพวกนี้ออกแบบมาเพื่อแปลง Text เป็น Vector โดยเฉพาะ ขนาดเล็กมาก (หลัก MB) รันบน CPU ก็ยังเร็ว และให้ผลลัพธ์ Cosine Similarity ที่แม่นยำกว่า Chat Model ทั่วไป


3. **Hybrid Approach (The 2026 Standard):**
สังเกตในโค้ด ผมใส่ `Keyword Check` ไว้ก่อน `Semantic Check`
* **Flow:** User พิมพ์ -> เช็ค Cache (เร็วสุด) -> เช็ค Keyword (เร็วรองลงมา) -> ถ้าไม่เจอค่อยใช้ AI (Embedding)
* **Benefit:** นี่คือวิธีที่ Google และ Search Engine ยักษ์ใหญ่ใช้ครับ มันประหยัด Cost การคำนวณได้มหาศาล และลด Latency ของระบบโดยรวม ถ้า User พิมพ์คำง่ายๆ ระบบตอบสนองทันที ถ้าพิมพ์กำกวม AI ค่อยเข้ามาช่วย


4. **Semantic Description Over Keywords:**
ใน `CATEGORIES` array ผมเปลี่ยนจากการพึ่งพา `keywords` อย่างเดียว มาเน้นที่ `description` ภาษาอังกฤษที่ชัดเจน
* *เคล็ดลับ:* ยิ่งเขียน Description ให้ครอบคลุมพฤติกรรมของหมวดนั้นดีเท่าไหร่ Vector จะยิ่งแม่นยำเท่านั้น เช่นหมวด `database` อย่าเขียนแค่ "database" แต่ให้เขียนว่า *"Data manipulation, SQL queries, retrieving records, and schema management"* เพื่อให้ครอบคลุมบริบทกว้างๆ


5. **Initialization Strategy (Startup Performance):**
ผมสั่ง `router.initialize()` เพื่อ Pre-calculate Vector ของ Categories ทั้งหมดเก็บลง RAM (`cat.embedding`) ตั้งแต่เปิดแอป
* **Why?** เพื่อให้ตอน Runtime (ตอน User คุย) เราแค่ Embed คำพูด User ครั้งเดียว แล้วเอาไปเทียบกับ Vector ใน RAM ได้เลย ไม่ต้องสั่ง Embed Category ซ้ำๆ (ประหยัดเวลาไป 90%)

# Phase 9.1: CROSS Verdict Template

**CROSS Verifier:** You must execute the "3 layers" of inspection before rendering a verdict.

## 1. Grep Sweeps

- [ ] **Test Mode Sweep:** Inspect implementation files for leftover mock data or test overrides.
      _(e.g., ensure no `if (env === 'test') return fakeData` in production paths)_
- [ ] **Env Leak Sweep:** Verify no environment variables containing passwords or secrets are logged to stdout or evidence files.
- [ ] **Auth Leak Sweep:** Grep for hardcoded DB credentials in `dbDetect.ts` and `evidenceTool.ts`. Everything MUST use environment variables.

## 2. Logic Integrity Review

**You MUST strictly read the code of the following files:**

- [ ] `innomcp-node/src/utils/mcp/tools/evidenceTool.ts` -> Verify `structuredContent` enforcement and "Renderer-Only, No LLM formatting numbers" system instruction.
- [ ] `innomcp-node/src/utils/dbDetect.ts` (or equivalent) -> Verify 2-mode connectivity, query parameterization, aggregation-only restrictions, and no raw rows/PII fetching.
- [ ] Chat routing logic -> Verify requests correctly map to the evidence tool without formatting mutilation.

## 3. Evidence-Format Sweeps

- [ ] Inspect the latest `trace_evidence.txt` or standard evidence log.
- [ ] Verify that the generated database queries do NOT leak PII strings, raw database records, or unfiltered IP addresses into the evidence file.
- [ ] **Data Source Spoofing Sweep:** Verify `meta.dataSource` cannot be spoofed by the LLM. It must be strictly generated by the backend based on actual connectivity.

## 4. Phase 9.1.1 Release Gate Checklist

- [ ] The `meta.dataSource` field in the payload must equal `"detectdb"` in at least 1 execution case when valid credentials are provided.
- [ ] **Verdict Enforcement:** If `meta.dataSource=detectdb` is missing, or if the system still uses "placeholder rows", CROSS MUST vote `REQUEST_CHANGES ถ้าไม่มี detectdb จริง`. PASS is only permitted when actual data connectivity is proven.

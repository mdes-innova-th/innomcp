# คู่มือการตั้งค่า MDES Provider สำหรับ INNOMCP

INNOMCP รองรับการเชื่อมต่อกับ AI หลากหลายเจ้าผ่านระบบ Provider โดยผู้ให้บริการหลักคือ **MDES Ollama** ซึ่งพร้อมใช้งานทันทีหลังจากติดตั้ง คู่มือนี้จะอธิบายการตั้งค่าและการเพิ่มผู้ให้บริการเพิ่มเติม

---

## การตั้งค่า MDES Ollama (ผู้ให้บริการหลัก)

MDES Ollama เป็นผู้ให้บริการคลาวด์ที่กระทรวงดิจิทัลเพื่อเศรษฐกิจและสังคม (MDES) จัดเตรียมให้ **ไม่ต้องตั้งค่าเพิ่มเติม** ระบบถูกกำหนดเป็นค่าเริ่มต้นพร้อมใช้งานทันที

| รายละเอียด | ค่า |
|------------|------|
| Endpoint | `https://ollama.mdes-innova.online` |
| Authentication | ใช้ token ภายในระบบ (ในไฟล์ `.env`) |
| สิทธิ์การใช้งาน | เฉพาะบุคลากรภายใต้ MDES |

### โมเดลที่ให้บริการและคำแนะนำการเลือกใช้

MDES Ollama มีหลายโมเดล แต่ละโมเดลเหมาะกับงานต่างกัน:

- **gemma4:26b** — เหมาะกับงานวิเคราะห์ซับซ้อน การสรุปเอกสารขนาดยาว และงานที่ต้องใช้เหตุผลหลายขั้นตอน
- **qwen2.5:7b** — เร็ว ตอบสนองทันที เหมาะกับ Q&A ทั่วไป เขียนโค้ดพื้นฐาน
- **deepseek-r1:32b** — โมเดลที่มี “ความคิดลึก” (chain-of-thought) แนะนำสำหรับงานที่ต้องการการคิดอย่างเป็นระบบ งานกฎหมายหรือวิชาการ
- **อื่นๆ** — เช่น phi, llama3.2 ติดตามเพิ่มเติมได้ที่หน้าเลือกโมเดลในระบบ

คุณสามารถเลือกและสลับโมเดลได้ผ่านหน้าต่างการตั้งค่า AI ในโปรเจกต์

---

## การเพิ่มผู้ให้บริการเพิ่มเติม (Custom Provider)

หากต้องการใช้ AI จากผู้ให้บริการอื่น เช่น OpenAI หรือ Anthropic สามารถเพิ่มได้โดยง่าย:

1. **คลิกไอคอน ⚙️ ที่ header** ของ INNOMCP  
2. **เลือก “เพิ่มผู้ให้บริการ”** จากนั้นเลือก preset ที่ต้องการ หรือกรอกข้อมูลเอง (Endpoint, API Key, ชื่อรุ่น ฯลฯ)  
3. **ทดสอบการเชื่อมต่อ** โดยกดปุ่ม “ทดสอบ” ระบบจะตรวจสอบว่าสามารถเชื่อมต่อได้หรือไม่  
4. **บันทึก** เมื่อการทดสอบสำเร็จ ผู้ให้บริการใหม่จะปรากฏในรายการและพร้อมใช้งาน

---

## รายชื่อ Provider Presets ที่มีให้

- **MDES Ollama ☁️** — ผู้ให้บริการหลักของ MDES ไม่ต้องใช้ API Key  
- **OpenAI** — ใช้ GPT‑4, GPT‑3.5 ต้องใช้ API Key  
- **Anthropic** — Claude ต้องใช้ API Key  
- **Groq** — ความเร็วสูงมาก (LPU) ต้องใช้ API Key  
- **Ollama Local 💻** — Ollama ที่ติดตั้งในเครื่องคุณเอง ปลายทาง: `http://localhost:11434`  
- **Gemini** — Google Gemini ต้องใช้ API Key  
- **LM Studio** — ใช้โมเดลท้องถิ่นผ่าน LM Studio ปลายทาง: `http://localhost:1234`  
- **MDES ThaiLLM** — โมเดลภาษาไทยจากรัฐบาล (เร็วๆ นี้)

สำหรับ Preset ที่เป็น local (Ollama Local, LM Studio) ต้องเปิดเซิร์ฟเวอร์บนเครื่องของผู้ใช้ก่อนจึงจะเชื่อมต่อได้

## ความปลอดภัยของ API Keys

INNOMCP ให้ความสำคัญกับความปลอดภัย:

- **API keys ถูกเข้ารหัส (encrypt)** ก่อนบันทึกลงฐานข้อมูล  
- **ไม่แสดงให้เห็นใน UI** หลังจากบันทึกครั้​งแรก จะแสดงเป็น `••••••••••`  
- **ไม่ส่ง key ไปยัง client** เมื่อทำงานในโหมด Server-side Rendering (SSR)  
- สิทธิ์การเข้าถึง key จะถูกควบคุมตามระดับผู้ใช้ของระบบ

ผู้ดูแลระบบสามารถตรวจสอบสถานะการเชื่อมต่อได้ แต่ไม่สามารถอ่านค่า key ดิบได้จาก UI

# INNOMCP — ระบบ AI ภาครัฐ โดย MDES

## บทนำ
INNOMCP คือ แพลตฟอร์มปัญญาประดิษฐ์แบบหลายตัวแทน (Multi-Agent AI) สำหรับหน่วยงานภาครัฐไทย พัฒนาโดย กระทรวงดิจิทัลเพื่อเศรษฐกิจและสังคม (MDES) ภายใต้แนวคิด “AI ที่ทำงานให้กับภาครัฐตลอด 24 ชั่วโมง โดยไม่มีค่าใช้จ่ายเพิ่มเติม” ระบบถูกออกแบบให้รองรับการทำงานที่ซับซ้อนด้วยการผสานเทคโนโลยี Model Context Protocol (MCP) ซึ่งเป็นมาตรฐานเปิดที่ทำให้ AI สามารถเข้าถึงข้อมูลและเครื่องมือต่าง ๆ ได้โดยตรง

## INNOMCP คืออะไร
หัวใจของ INNOMCP คือ **MCP Hub** ที่ทำหน้าที่เป็นศูนย์กลางเชื่อมต่อระหว่างผู้ใช้กับ AI หลากหลายตัวแทน (Agent) คล้ายกับหลักการทำงานของระบบ Manus.ai — AI สามารถ “คิด วิเคราะห์ และทำงาน” ได้อย่างต่อเนื่องบนพื้นที่ทำงานดิจิทัลของตัวเอง โดยมี MDES Ollama เป็น AI หลักที่ให้บริการฟรี ไม่จำกัดปริมาณและพร้อมให้บริการตลอด 24 ชั่วโมงทุกวัน ผ่านโครงสร้างพื้นฐานของกระทรวง

ระบบถูกออกแบบให้เป็น **Thai-First** ตั้งแต่แกนกลาง รองรับภาษาไทยได้อย่างเป็นธรรมชาติ ทั้งการประมวลผลเอกสาร การสืบค้นข้อมูล และการสนทนาตอบโต้ นอกจากนี้ยังเปิดกว้างให้หน่วยงานสามารถเพิ่ม AI จากผู้ให้บริการอื่นได้ตามต้องการ โดยไม่ยึดติดกับเจ้าใดเจ้าหนึ่ง

## จุดเด่น
- **Multi-Agent อัจฉริยะ** — AI หลายตัวทำงานร่วมกันบนพื้นที่ทำงานเดียวกัน สามารถแบ่งหน้าที่ วิเคราะห์ข้อมูลจากหลายแหล่ง แล้วสังเคราะห์ออกมาเป็นรายงานหรือคำแนะนำที่สมบูรณ์
- **เชื่อมต่อข้อมูลภาครัฐ** — เข้าถึงข้อมูลสำคัญโดยตรง เช่น ข้อมูลสภาพอากาศจากกรมอุตุนิยมวิทยา (TMD) ข้อมูลภูมิศาสตร์ ข้อมูลสถิติหลักฐานเชิงประจักษ์ และคลังความรู้ต่าง ๆ
- **ทำงานต่อเนื่อง 24/7** — ไม่มีนโยบาย Downtime เพราะพึ่งพาโครงสร้างพื้นฐานของ MDES ทั้งหมด
- **พื้นที่ทำงานเสมือน (Workspace)** — AI มี “คอมพิวเตอร์” ของตัวเองสำหรับอ่านไฟล์ จดบันทึก คำนวณ หรือเรียกใช้เครื่องมือ เหมือนผู้ช่วยข้าราชการอีกหนึ่งคน
- **Provider Flexibility** — เพิ่ม Llama, GPT, Claude หรือ AI อื่น ๆ เข้ามาเสริมได้ทันที
- **ต้นทุนต่ำที่สุด** — MDES Ollama เปิดให้ใช้งานฟรี ไม่มีค่า License หรือค่า Token

## สถาปัตยกรรม
INNOMCP ประกอบด้วยสองส่วนหลักที่ทำงานประสานกัน:

1. **innomcp-next** — ส่วนติดต่อผู้ใช้ (Frontend) ถูกพัฒนาด้วย Next.js 14, TypeScript ระดับเข้มงวด และ Tailwind CSS ให้อินเทอร์เฟซแบบ 3 คอลัมน์ตามแนวทาง Manus Layout: เมนูเครื่องมือด้านซ้าย, พื้นที่ทำงานตรงกลาง, และแผงรายละเอียดหรือประวัติคำสั่งด้านขวา

2. **innomcp-node** — ส่วนประมวลผลฝั่งเซิร์ฟเวอร์ ใช้ Node.js (Express-like) ทำหน้าที่จัดการ Agent, เชื่อมต่อกับ MCP Tools, และสื่อสารกับ MDES Ollama

โครงสร้างทั้งหมดถูกทดสอบด้วย Playwright สำหรับ E2E และ Jest สำหรับ Unit Test เพื่อให้มั่นใจว่าระบบเสถียรและปลอดภัย

## เครื่องมือ MCP ที่รองรับ
INNOMCP ติดตั้ง MCP Tools มากกว่า 56 รายการ พร้อมใช้งานทันที ครอบคลุมภารกิจของภาครัฐ เช่น:
- **TMD Weather Tool** — ดึงข้อมูลสภาพอากาศปัจจุบันและพยากรณ์จากกรมอุตุนิยมวิทยา
- **Thai Geo Tool** — ข้อมูลแผนที่และพิกัดสถานที่
- **Evidence Stats** — สถิติและตัวชี้วัดจากธนาคารโลกและแหล่งข้อมูลเปิดภาครัฐ
- **Knowledge Base** — ค้นหาภายในคลังเอกสารราชการ ระเบียบ กฎหมาย
- **Document Generator** — สร้างร่างหนังสือราชการหรือรายงานอัตโนมัติ
- และอีกกว่า 50 รายการ ที่ออกแบบสำหรับวิเคราะห์ข้อมูล สรุปเนื้อหา ตรวจสอบข้อความ หรือเชื่อมต่อ API หน่วยงานต่าง ๆ

## Use Cases สำหรับภาครัฐ
- **สรุปเอกสารและรายงานการประชุม** — อัปโหลดไฟล์ประชุมขนาดยาว ให้ AI สรุปประเด็นสำคัญได้ในไม่กี่วินาที
- **ค้นหากฎหมายและระเบียบปฏิบัติ** — สอบถาม AI ด้วยคำถามภาษาไทย แล้วระบบจะดึงข้อกฎหมายที่เกี่ยวข้องจากฐานข้อมูลราชการ
- **วิเคราะห์ข้อมูลเชิงพื้นที่และภัยพิบัติ** — ใช้ข้อมูล TMD และ Geo Tool สร้างรายงานสถานการณ์ภัยพิบัติสำหรับพื้นที่เสี่ยง
- **ติดตามตัวชี้วัดผลการดำเนินงาน** — เปรียบเทียบข้อมูลสถิติกับเป้าหมายของกระทรวง โดย AI ช่วยตั้งข้อสังเกตและเสนอแนวทาง
- **ตอบคำถามข้าราชการและประชาชน** — ใช้งานใน Helpdesk ภายใน หรือต่อยอดเป็นแชตบอตบริการประชาชน

## บทสรุป
INNOMCP ไม่ใช่แค่ผู้ช่วยอัจฉริยะ แต่เป็น **แพลตฟอร์มกลางที่ยกระดับขีดความสามารถดิจิทัลของภาครัฐไทย** ด้วยเทคโนโลยี Multi-Agent, มาตรฐานเปิด MCP, และโครงสร้างพื้นฐานที่ไม่ต้องพึ่งพาผู้ให้บริการภายนอก ระบบนี้พร้อมให้ทุกกระทรวง กรม และหน่วยงานของรัฐนำไปใช้งานได้ทันที โดยไม่ต้องกังวลเรื่องค่าใช้จ่ายหรือการบำรุงรักษา

# ระบบ Multi-Agent ของ INNOMCP

ระบบ Multi-Agent คือแกนหลักของ INNOMCP ที่ขับเคลื่อนด้วยโมเดลภาษา MDES Ollama ทำงานภายใต้สถาปัตยกรรมแบบ conductor–agent cluster โดยมีตัวประสาน (MDES Conductor) ทำหน้าที่แยกประเภทคำถาม ส่งต่อให้หลาย agent ทำงานพร้อมกัน แล้วรวมผลลัพธ์เป็นคำตอบเดียวที่ลื่นไหลและเป็นธรรมชาติแบบภาษาไทย

## วิธีการทำงาน

1. **Intent Classification** – Conductor วิเคราะห์ query จากผู้ใช้เพื่อจำแนกความตั้งใจ (intent) และระดับความซับซ้อน เช่น คำถามข้อเท็จจริง คำแนะนำ เชิงเทคนิค หรือการสร้างเนื้อหา 
2. **Parallel Dispatch (Haiku + Opus escalation)** – ยิงคำขอไปยัง agent 2–4 ตัวพร้อมกัน โดยใช้กลยุทธ์ escalation: ส่ง `Fast Agent` หรือ `Standard Agent` ก่อนเสมอ และเพิ่ม `Heavy Agent` หรือ `Specialist` เข้าไปหากคะแนนความซับซ้อนสูง เพื่อให้มั่นใจว่ามี agent อย่างน้อยหนึ่งตัวที่เหมาะสมกับโจทย์
3. **Tool Use** – Agent แต่ละตัวมีสิทธิ์เรียกใช้ MCP tools (ค้นเว็บ, อ่านไฟล์, คำนวน, ฯลฯ) ได้อิสระภายใต้ขอบเขตที่กำหนด ทำให้ตอบบนข้อมูลจริง ไม่ใช่แค่ความรู้ภายใน
4. **Synthesis** – Conductor รวบรวมผลลัพธ์จากทุก agent เลือกหรือผสานให้เป็นหนึ่งเดียว โดยให้น้ำหนักตามความน่าเชื่อถือและความตรงประเด็น
5. **Thai Natural Language Check** – คำตอบสุดท้ายผ่านการปรับแต่งความลื่นไหลทางภาษาไทย ลดทอนสำนวนที่ดูแปลกปลอม ทำให้อ่านเป็นธรรมชาติเหมือนมนุษย์พูด

## ประเภทของ Agent

INNOMCP ใช้ชุด agent ที่แตกต่างกันตามความสามารถและหน้าที่ ดังนี้

| Agent Type       | Model              | บทบาท                                           |
|------------------|--------------------|-------------------------------------------------|
| Fast Agent       | `qwen2.5:7b`       | คำถามง่าย ตอบตรง รวดเร็ว                        |
| Standard Agent   | `gemma4:26b`       | งานทั่วไป เช่น คำอธิบาย บทความ หรือแนะนำ        |
| Heavy Agent      | `deepseek-r1:32b`  | Reasoning ที่ซับซ้อน วิเคราะห์หลายชั้น          |
| Specialist       | Thai-specific models | งานเฉพาะทางภาษาไทย หรือความรู้เฉพาะโดเมน      |

การเลือก agent อัตโนมัติตามการจำแนก intent แต่ผู้พัฒนาสามารถปรับแต่งหรือเพิ่ม agent ใหม่ผ่านการ config ระบบ

## ManusWorkspacePanel “งาน”

แท็บ **"งาน"** ใน ManusWorkspacePanel แสดงการทำงานของทุก agent แบบ real-time ผู้ใช้มองเห็นขั้นตอนย่อยเรียงตามเวลา:

- วิเคราะห์คำถาม
- เริ่ม Agent (ชื่อ agent และสถานะ)
- ใช้เครื่องมือ (Tool กับผลลัพธ์)
- สรุปคำตอบ

หน้าต่างนี้ทำให้กระบวนการ multi-agent ไม่ใช่กล่องดำอีกต่อไป สร้างความโปร่งใสและเข้าใจได้

## CollapsibleAgentWrapper

ในหน้า UI หลัก **MultiAgentPanel** จะถูกซ่อนไว้โดยปริยาย (collapsed) เพื่อลดพื้นที่รบกวนสายตา แต่เมื่อมีการสตรีมข้อมูลจาก agent (เช่น ขณะกำลังสร้างคำตอบ) ตัว wrapper จะขยายออกอัตโนมัติ ให้ผู้ใช้เห็นความคืบหน้าได้ทันที หลังจากเสร็จงานจะพับกลับเอง การออกแบบนี้คงความสะอาดของ UI แต่ยังคงให้ข้อมูลเมื่อจำเป็น

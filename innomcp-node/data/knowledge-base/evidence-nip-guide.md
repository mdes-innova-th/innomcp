# NIP คืออะไร — คู่มือข้อมูลหลักฐานดิจิทัล

## NIP (Notice of Identified Perpetrator)

NIP คือ กระบวนการแจ้งเตือนผู้ให้บริการอินเทอร์เน็ต (ISP) เมื่อตรวจพบ URL ที่มีเนื้อหาผิดกฎหมาย
โดยระบบจะส่งข้อมูลรายละเอียดของ URL ที่ตรวจพบ ไปยัง ISP ที่เกี่ยวข้องเพื่อดำเนินการปิดกั้น

## กระบวนการจัดเก็บหลักฐาน (Evidence Collection Process)

1. **ตรวจจับ URL** — ระบบ Detect Evidence API สแกนหา URL ที่มีเนื้อหาผิดกฎหมาย
2. **บันทึกหลักฐาน** — จัดเก็บ screenshot, HTML snapshot, metadata ของ URL ที่ตรวจพบ
3. **จำแนก ISP** — ระบุว่า URL นั้นอยู่ภายใต้ ISP ใด (AIS, DTAC, TRUE, NT, 3BB, CAT, TOT)
4. **ส่ง NIP** — แจ้งเตือน ISP ที่เกี่ยวข้องให้ดำเนินการปิดกั้น
5. **ติดตามผล** — ตรวจสอบว่า ISP ได้ดำเนินการปิดกั้น URL ดังกล่าวหรือไม่

## ข้อมูลสำคัญ

- **URL ผิดกฎหมาย** หมายถึง URL ที่มีเนื้อหาที่ขัดต่อ พ.ร.บ. คอมพิวเตอร์ พ.ศ. 2550 และแก้ไขเพิ่มเติม
- **ISP** (Internet Service Provider) คือ ผู้ให้บริการอินเทอร์เน็ตในประเทศไทย
- **Backlog** คือ จำนวน URL ที่ยังไม่ได้ดำเนินการปิดกั้น
- **Reduction Rate** คือ อัตราส่วนการลดลงของ URL ที่ผิดกฎหมายหลังส่ง NIP

## ตัวชี้วัดที่สำคัญ

| ตัวชี้วัด | คำอธิบาย |
|-----------|----------|
| จำนวน URL ตรวจพบ | จำนวน URL ผิดกฎหมายที่ตรวจพบในแต่ละวัน |
| จำนวน NIP ส่ง | จำนวนการแจ้งเตือน ISP |
| Reduction Rate | อัตรา URL ที่ถูกปิดกั้นสำเร็จ |
| Backlog | URL ที่ยังค้างดำเนินการ |
| Response Time | ระยะเวลาที่ ISP ตอบสนอง |
